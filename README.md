## Project Documentation

### 1. Project Overview

#### 1.1 Introduction

Wheel-n-Deal is a collaborative commuting and package delivery platform that aims to revolutionize daily commuting by integrating package deliveries into users' daily journeys.

#### 1.2 Objectives

The main objectives of Wheel-n-Deal include optimizing commute efficiency, providing a platform for commuters to earn extra income, and contributing to a sustainable and efficient delivery ecosystem.

#### 1.3 Stakeholders

- **Users (Commuters):** Individuals who commute daily and are interested in earning extra income by delivering packages.
- **Package Senders and Recipients:** Users who send or receive packages using the Wheel-n-Deal platform.

#### 3.2 Requirement Analysis

Requirement analysis involves gathering, documenting, and validating the needs and expectations for the system. Key aspects include understanding needs, gathering information, analysis and prioritization, documentation, validation and verification, and an iterative process.

##### 3.2.2 Requirement Analysis Techniques

1. **Interviews:** Conduct structured discussions with stakeholders to elicit detailed information.
2. **Surveys and Questionnaires:** Distribute structured sets of questions to gather opinions and preferences.
3. **Workshops and Brainstorming:** Facilitate collaborative sessions to generate ideas and clarify requirements.
4. **Prototyping:** Develop preliminary models to gather feedback before full-scale development.
5. **Use Case Analysis:** Identify functional requirements through scenarios of system-user interactions.
6. **Observation and Ethnographic Studies:** Observe users in their natural environment to understand behaviors and needs.
7. **Requirement Workshops:** Organize structured sessions to collaboratively define, refine, and prioritize requirements.
8. **Document Analysis:** Review existing documentation to extract relevant requirements.
